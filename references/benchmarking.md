# 同类竞争审计方法论

**When to read**: Phase 3 执行 C 维度（同类竞争审计）时。本文件定义 C 维度同类竞争审计的完整方法论。

---

## Step C1: SkillHub 搜索与排名

### API 调用

```
URL: https://api.skillhub.cn/api/v1/search?q=<keywords>
Keywords: extract from the created Skill's name + core function
Example: for "interview-insight-miner", search "interview insight analysis"
```

API 返回质量指标：`downloads`, `installs`, `stars`, `score`。

### 质量排序公式

```
综合评分 = downloads × 0.4 + installs × 0.3 + stars × 0.3
```

| Metric | Weight | Rationale |
|--------|--------|-----------|
| downloads | 0.4 | 最客观 — 下载量反映真实价值 |
| installs | 0.3 | 更精确 — 下载后实际安装 |
| stars | 0.3 | 社区认可 — 但受年龄影响（老 Skill 星多） |

按分数降序排序，取 **Top 3**。

### CLI Fallback（API 不可用时）

```bash
skillhub search <keywords>
```

CLI 不返回 downloads/stars，用代理排名：
1. 版本号高 → 迭代多 → 质量可能更好
2. description 长（>200 字符）→ 作者更认真
3. description 含具体触发场景 → 更实用

取 Top 3。

### 网络不可用降级

- API 和 CLI 都不可达 → C 维度标注"网络不可用，未做同类比对"
- 其他维度正常执行
- 综合分计算时 C 维度按 7.0（中性分）计算

## Step C2: 腾讯手册 9 维度比对

与 Top 3 同类按 **9 个腾讯技能手册维度** 比对：

| # | 腾讯原则 | 检查内容 |
|---|---------|---------|
| 1 | Description: 触发精准度 | description 是否明确说明何时调用？ |
| 2 | Description: 关键词前置 | 核心触发词是否在前 200 字符？ |
| 3 | Description: Do NOT 范围 | description 是否明确声明不做什么？ |
| 4 | 一 Skill 一职 | 是否聚焦单一场景 + 单一交付物？ |

## Step C3: 差异化分析

### 如果被审计 Skill 与已有高质量 Skill 重复

- 重复度 > 70% → 建议 `skillhub install <slug>` 安装已有
- 重复度 40-70% → 列出差异化，由用户决定
- 重复度 < 40% → 明确差异化定位

### 如果有意义的差异

记录差异化：
```
vs [Peer #1 name]: 我们做X不做Y，对方做Y不做X
vs [Peer #2 name]: 我们的输出格式更[具体/专业/简洁]
vs [Peer #3 name]: 我们的规则覆盖了[对方遗漏的边界情况]
```

### 差异化维度

| 维度 | 比较点 |
|------|--------|
| 功能范围 | 做 X 不做 Y vs 对方做 Y 不做 X |
| 输出格式 | 固定表格 vs 自由文本 |
| 触发条件 | 精准触发 vs 模糊匹配 |
| 平台支持 | 国内优先 vs 海外优先 |
| 语言 | 中文优先 vs 英文优先 |
| 集成 | 飞书/IMA/Obsidian vs 无集成 |

## Step C4: 盲区识别

如果 Top 3 揭示了被审计 Skill 的盲区，列出具体改进：

```
盲区1: [具体问题]（违反腾讯手册原则：[哪条原则]）
  → 修复方案：[具体修复]
  → 严重性：Important
盲区2: [具体问题]（违反腾讯手册原则：[哪条原则]）
  → 修复方案：[具体修复]
  → 严重性：Minor
```

**常见盲区**：
- 示例未覆盖中文场景
- Do NOT 范围过窄（漏排相邻领域）
- 输出格式含"整理清晰"等模糊指令
- 规则含"适当处理"等不可执行规则

---

## Step C5: 重复判定与建议

| 重复度 | 判定 | 建议 | 严重性 |
|--------|------|------|--------|
| > 70% | 重复 | 建议安装已有 Skill | Important |
| 40-70% | 部分重叠 | 明确差异化定位 | FYI |
| < 40% | 独立 | 无需行动 | FYI |

**重复度评估方法**：
1. 核心 trigger keywords 重叠率
2. description 三要素（做什么/何时触发/Do NOT）重叠率
3. 输出格式相似度
4. 功能范围交集占比

---

## C 维度评分标准

| 评分 | 标准 |
|------|------|
| 9-10 | 独立定位 + 腾讯9维度全优 + 无盲区 |
| 7-8 | 有差异化 + 9维度良好 + 少量盲区（Minor） |
| 5-6 | 部分重叠 + 9维度及格 + 有 Important 盲区 |
| 3-4 | 重叠度高 + 9维度不及格 + 多个 Important 盲区 |
| 0-2 | 完全重复 + 9维度严重缺失 |

**网络不可用时**：C 维度按 7.0（中性分）计算，标注"未做同类比对"。
